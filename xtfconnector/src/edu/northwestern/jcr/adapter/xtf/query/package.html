Implements a Jackrabbit query handler using XTF and other query handling classes.

Both the XPath and SQL queries are translated by the Jackrabbit core to the Abstract Query Tree (AQT), the common query description format that allows Jackrabbit to implement a query engine which is (to a certain extent) independent of the query syntax used (XPath or SQL). The "Search Implementation" section of the Jackrabbit website details the implementation of searches in Jackrabbit along with the default search index being used.

The search index in Jackrabbit is pluggable and has a default implementation based on Apache Lucene. The default implementation is independent of the persistence manager being used. To connect to an existing XTF repository, a new search index manager needs to be written to interpret the AQT and generate raw XML query statements that can be executed against the raw servlet of the XTF repository.
